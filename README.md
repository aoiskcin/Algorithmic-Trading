# Algorithmic S&P 500 Equal-Weight Portfolio Builder

An automated asset allocation and financial data pipeline built in Python. This application extracts real-time stock metrics for all active S&P 500 constituents and dynamically structures an optimized, equal-weight investment portfolio exported directly into a customized executive spreadsheet.

## Architecture & Tech Stack

* **Language:** Python
* **Data Engineering Libraries:** Pandas, NumPy
* **Financial Data Engine:** yfinance API (Synchronous Data Pipeline)
* **Spreadsheet Automation Engine:** XlsxWriter
* **Core Concepts:** Data ingestion, asynchronous network fault debugging, vector-based row staging, format styling blocks, programmatic data validation bounds.

## Engineering Challenges & Optimization

### 1. Legacy API Deprecation & Refactoring
The original structural dependency of this financial module relied on legacy stock data endpoints that have since faced full retirement. Rather than abandoning the architecture, I audited the network failures, traced the execution bottlenecks, and successfully refactored the entire ingestion pipeline to stream real-time prices and market caps securely via alternative financial APIs.

### 2. Streamlining Network Latency
The baseline implementation required sequential single-ticker requests, running into heavy network I/O blockades and a 1-to-2 minute execution cycle. To simulate corporate-grade software standards, I restructured the module to isolate structural data vectors using lists, which significantly lowered server traffic overhead and cleaned up execution runtime logs.

### 3. Data Sanitization Boundaries
Real-world market data fluctuates constantly with corporate merges, delistings, and structural symbol changes (e.g., handling ticker formats like `BRK.B`). I integrated structured Exception Handling blocks to catch network errors (`HTTP Error 404`) and isolate data missing states (`NaN` values) natively, preventing application runtime crashes during heavy calculation loops.

## Code Structure & Operational Pipeline

1. **Data Acquisition:** Reads a standardized, local baseline listing of index constituents and initiates a loop array across active stock endpoints.
2. **Mathematical Equal-Weighting Calculation:** Accepts a dynamic user input valuation (e.g., $1,000,000), evaluates the active index pool size, partitions individual cash budgets per stock, and floors the values to integer-based whole shares.
3. **Automated Excel Compiling:** Establishes an XlsxWriter workbook data pipeline, overrides basic spreadsheet defaults with distinct header color arrays, applies currency/numerical cell formats, and closes data buffers cleanly to prevent operating system file-lock conflicts.
